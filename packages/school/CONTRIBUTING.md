# Guia de Contribuição

Obrigado pelo seu interesse em contribuir para o School Manager System! Este documento descreve as diretrizes para contribuir com este projeto.

## Como Contribuir

1.  **Fork o repositório**: Crie uma cópia do repositório em sua conta.
2.  **Crie uma branch**: Use uma branch separada para suas alterações (`git checkout -b feature/nova-feature`).
3.  **Faça suas alterações**: Implemente suas melhorias ou correções.
4.  **Teste suas alterações**: Certifique-se de que tudo funciona como esperado.
5.  **Envie um Pull Request**: Descreva suas alterações detalhadamente.

## Padrões de Código

-   Siga as convenções de estilo do Dart e Flutter.
-   Respeite as regras de lint definidas no arquivo `analysis_options.yaml` do projeto.
-   Documente classes e métodos públicos e use `/// Doc comments` para documentação de API.
-   Mantenha a estrutura de pastas organizada conforme o padrão do pacote/módulo.
-   Escreva testes para novas funcionalidades sempre que possível.

## Reportando Problemas

-   Use a aba "Issues" para relatar bugs ou sugerir novas funcionalidades.
-   Seja claro e forneça passos para reproduzir o problema.
-   Inclua logs, trechos de código ou screenshots se possível.

## Commit Messages

-   Use mensagens de commit claras e descritivas (em inglês, preferencialmente seguindo o padrão Conventional Commits).
-   Exemplo: `feat: add user authentication` ou `fix: resolve login error`.

## Licença

Ao contribuir, você concorda que suas contribuições serão licenciadas sob a licença do projeto (veja o arquivo LICENSE).
